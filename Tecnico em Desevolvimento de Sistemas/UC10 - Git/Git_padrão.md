# 📦 Padrões de Commit — Guia Visual

> “Código sem padrão é igual commit sem sentido: ninguém entende, ninguém confia.”

---

## ✅ Estrutura Básica
<tipo>: <descrição curta>


---

## 🧠 Tabela de Tipos

| Tipo       | Quando usar                                                                 |
|------------|------------------------------------------------------------------------------|
| `feat`     | Nova funcionalidade                                                          |
| `fix`      | Correção de bug                                                              |
| `docs`     | Mudanças na documentação                                                     |
| `style`    | Formatação, espaços, pontuação (sem mudar código)                            |
| `refactor` | Mudança no código que não altera comportamento                               |
| `test`     | Adicionar ou corrigir testes                                                 |
| `chore`    | Tarefas gerais: build, dependências, configs                                 |

---

## 🧪 Exemplos Práticos

- eat: adiciona botão de recuperar senha
- fix: corrige erro ao salvar usuário sem email
- docs: atualiza README com instruções de instalação
- style: remove espaços extras no arquivo main.css
- refactor: simplifica lógica de validação de CPF
- test: adiciona teste para cálculo de frete
- chore: atualiza versão do React para 18.2.0
